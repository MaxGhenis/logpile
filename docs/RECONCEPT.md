# Logpile reconcept — record, not dashboard

*2026-07-11. Context: launch week. The July landscape scan settled where Logpile can win, and the web app (built spring 2026) predates that conclusion.*

## The problem with the current site

The home page is an analytics dashboard: four stat cards, four charts, a recent-sessions table. That is the most crowded lane in the ecosystem — ccusage (17k stars), codeburn (8.6k), tokscale (4.4k), claude-devtools (3.7k) all sell "see your usage." Logpile's differentiated assets — transcripts as readable artifacts, the secret-scanning publish queue, operator profiles — sit behind nav tabs. A visitor in public mode lands on numbers *about* the operator instead of work *by* the operator.

## The thesis

**Logpile is the record of agentic work.** Sessions are work product, like commits and PRs — worth keeping, reading, citing, and (selectively) publishing as proof of how you operate. Analytics is a supporting lens on the record, not the product.

Straude answers "how much." Logpile answers "what, exactly — with receipts."

## Object model

| Object | Is | Analogy |
|---|---|---|
| Session | The atomic artifact: transcript + what it did (writes, tests, commits) + provenance (tool, model, tokens, origin) | a PR |
| Day | A ledger entry: the day's sessions and totals | a changelog entry |
| Operator | A portfolio: published sessions + activity shape | a GitHub profile |
| Publish review | The gate between private record and public proof (secret/PII scan, redaction) | code review |

## Surface map

| Route | Was | Becomes | Why |
|---|---|---|---|
| `/` | Dashboard (stats + charts) | **Record** — day-grouped journal of sessions with status/activity badges; compact ledger strip; charts demoted below the fold | The daily question is "what did my agents do, and where is it," not "how many tokens" |
| `/sessions` | Archive with filters | unchanged | Already the right shape |
| `/repos` | Repo list | unchanged | |
| `/u` | People | **Operators** — portfolio emphasis; phase 2 adds pinned sessions and a proof strip | The hire-ability surface |
| `/analysis` | Analysis | **Ledger** — absorbs the analytics; corrected token accounting is first-class (replay dedup, cache economics) | One honest place for numbers |
| `/publish` | Publish queue | unchanged | The moat; keep gate language |

Home charts: activity + CC-vs-Codex stay on `/` below the record; top-tools and errors-by-operator move to the Ledger (they were removed from home in this pass and should be added to `/analysis` when it becomes Ledger).

## Public mode / logpile.ai

- Now: static landing at logpile.ai (same charred-wood identity) — thesis hero over a live ledger of real daily aggregates, install, three pillars (keep → read → publish), local-first trust line. Generated by `scripts/build_landing.py`, deployed as a Cloudflare Worker.
- Phase 2 (post-September gate): logpile.ai serves the app in public mode, `/` renders the landing with live public sessions, `/u/<name>` profiles become the demo. Requires the publish-queue privacy audit first.

## Voice

Direct, numbers over adjectives, sentence case. Tagline: **"the record of agentic work"** (replaces "stacked session memory"). Trust line appears wherever install does: *local-first; nothing leaves your machine unless you publish it.*

## Aesthetic

Unchanged on purpose: warm amber on charred wood, Playfair Display / Plus Jakarta Sans / JetBrains Mono, grain overlay, stacked-bars mark. The identity was already right; the information architecture wasn't.
